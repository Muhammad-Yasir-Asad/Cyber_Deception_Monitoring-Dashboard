## Feature Implementation Status

### Feature 1: Socket.io Real-Time Push ✅ **IMPLEMENTED**
**File:** `server/socket/liveEvents.js`

**What it does:**
- Polls attack_logs every 3 seconds → emits `new_attack` events to all clients
- Polls honeytoken_alerts every 5 seconds → emits `honeytoken_triggered` events
- Polls stats every 10 seconds → emits `stats_update` events
- Establishes fallback polling + WebSocket connections for real-time data

**Integration:**
- Server: Attached to HTTP server in `server.js` via `initSocket(server)`
- Client: Connected in `App.js` with socket handlers for all 3 event types
- Honeytoken triggered banner shows with 10-second auto-dismiss

### Feature 3: Block IP Route ✅ **IMPLEMENTED**
**File:** `server/routes/block.route.js`

**Endpoints:**
- `POST /api/block/:ip` — Block an IP address using iptables
- `GET /api/block/status/:ip` — Check if IP is blocked

**Implementation:**
- IPv4 validation via regex: `/^(\d{1,3}\.){3}\d{1,3}$/`
- Executes iptables commands to drop INPUT and OUTPUT traffic
- Returns `{ success: true, ip, blockedAt }` on success
- Requires root/sudo privileges to execute iptables
- Graceful error handling: Returns "Insufficient permissions — server must run as root"

**Wired UI:**
- "⊘ Block" button in AttackerProfiles table
- Shows loading state while blocking
- Updates UI to "Blocked" status with reduced opacity

**Status:** Production-ready (requires root for actual blocking) ✅

### Feature 7: Enhanced Threat Scoring Service ✅ **IMPLEMENTED**
**File:** `server/services/threatScoring.service.js`

**Exports:**
1. `calculateThreatScore(profile, recentAttacks)` — Returns 0–100 score
   - **Severity Factor** (max 30pts): CRITICAL=4, HIGH=3, MEDIUM=2, LOW=1 per event
   - **Attack Type Factor** (max 20pts): SQLi×4, Traversal×3, Bruteforce×3, XSS×2, CSRF×2, IDOR×2
   - **Persistence Factor** (max 30pts): >7 days=30, >1 day=15, else=0
   - **Velocity Factor** (max 10pts): >5 req/min=10, >1 req/min=5, else=2
   - **Reputation Factor** (max 10pts): +10 if is_known_malicious
   - **Bonus Flags** (+5 each): attempted_account_takeover, attempted_funds_transfer, attempted_privilege_escalation

2. `generateThreatScorecard(profile, attacks)` — Detailed breakdown
   - Returns: total, level (CRITICAL/HIGH/MEDIUM/LOW), breakdown { severity, attackType, persistence, velocity, reputation }

3. `generateLeaderboard(profiles, attackMap)` — Ranked threat list
   - Maps each profile through calculateThreatScore
   - Sorts by calculatedScore DESC
   - Returns ranked array with attackers + their calculated scores

**Integration Points:**
- Used in `stats.route.js` to calculate topAttackersDetailed with calculatedThreatScore
- Used in `socket/liveEvents.js` for real-time stats updates
- Supplements database threat_score field with dynamic calculation

**Status:** Production-ready ✅

### Issue #2: SQL Query Severity Filtering ❌ → ✅ **FIXED**
**Severity:** HIGH  
**Files:** 
- `server/socket/liveEvents.js` (lines 36–46, 79–83)
- `server/routes/stats.route.js` (lines 33–44, 84–89)

**Problem:** 
- Query selected `severity` column and tried to match against string values ('CRITICAL', 'HIGH', etc.)
- Schema defines `severity` as INTEGER 1–10, not VARCHAR
- Database provides a computed column `severity_label` for the string values

**Fix:**
1. Changed `SELECT severity` → `SELECT severity_label`
2. Updated getCount logic: `r.severity === level` → `r.severity_label === level`

**Impact:** Stats and socket events now correctly group attacks by severity level  
**Files Modified:** 2 (liveEvents.js, stats.route.js)
